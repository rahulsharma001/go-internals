# Application targets — practice vs next phase

> **Not a job board.** This is a **phased target list** for *how* to sequence applications. Names are **examples** of employers often discussed for backend / Go-heavy work — **verify open roles, location, and bar on the company’s careers site** before you invest time.
>
> **Pair with:** [[Study Plan]] application gates (Gate 0 / 1 / 2) so “practice” vs “serious” loops match your evidence, not this table alone.

---

## How to use the phases

| Phase | Intent | When (see [[Study Plan]]) |
|-------|--------|---------------------------|
| **P0 — Calibration** | Build interview muscle, log real questions, tolerate rejections | Around **Gate 0** |
| **P1 — Target** | Strong Go / backend scope; comp and process vary | After **Gate 1** evidence + mocks |
| **P2 — Stretch** | Highest bar or hottest brands; apply when P1 feedback is strong | After **Gate 2** or selective parallel |

Sort **within** each phase by your own constraints (visa, remote, domain). Below, names are ordered roughly **P0 → P1 → P2** as *typical* sequencing ideas from public interview chatter — **not** a ranking of “better company.”

---

## P0 — Calibration / volume (examples)

Good for **early reps** and logging what gets asked. Replace with **your** local high-volume employers if these don’t apply.

- Fast-growing **product** or **services** shops with active hiring (check LinkedIn / Naukri / Wellfound).
- **Referrals** at non-dream employers (often fastest calibration).
- India-focused examples often mentioned in forums (verify roles): **Razorpay**, **CRED**, **Zerodha**, **Juspay** — *only if they list relevant backend/Go openings.*

---

## P1 — Strong backend / Go signal (examples)

Often cited in senior backend / distributed systems discussions. **Confirm Go is in the job description** — many teams are Java/C++ multi-language.

| Example | Notes (illustrative) |
|---------|----------------------|
| **Uber** | Frequently cited for real-time / high-QPS backend; multi-language — check Go specifically. |
| **Stripe** | Payments correctness + distributed systems; deep interviews — often **after** you’re warmed up. |
| **Datadog** | Observability / agents / ingestion; Go common — verify team. |
| **HashiCorp** | Go-centric product suite — narrower hiring windows. |
| **Coinbase** | Low-latency / trading infra — verify Go vs other stacks. |
| **Airbnb** | Mixed stack — confirm backend role language. |

---

## P2 — Stretch / FAANG-adjacent (examples)

Higher variance in **language** (Go may be infra/tooling only). Use when P1 mocks feel solid.

| Example | Notes (illustrative) |
|---------|----------------------|
| **Google** | Go in cloud/k8s/tooling areas — role-dependent. |
| **Amazon** | Many roles are Java — filter for Go explicitly. |
| **Meta** | Often **not** primary Go — infra exceptions exist. |
| **Netflix / Apple** | Mixed; Go often infra-adjacent — read JD carefully. |
